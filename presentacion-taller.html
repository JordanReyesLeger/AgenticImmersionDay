💡 Concepto Clave

¿Qué es un Agente de IA?

Un agente es un asistente inteligente con personalidad (instrucciones), cerebro (modelo LLM) y superpoderes (herramientas). Entiende preguntas, decide qué herramienta usar y responde con información contextual.

  • ✓ Entiende lenguaje natural del usuario
  • ✓ Decide qué herramienta usar automáticamente
  • ✓ Ejecuta acciones: código, búsquedas, análisis
  • ✓ Responde con datos reales y citaciones

Foundry Agents + Agent Framework SDK

El ecosistema de agentes de Microsoft tiene dos piezas clave que trabajan juntas:

☁️

Foundry Agents

Es el servicio en la nube dentro de Azure AI Foundry que hospeda, ejecuta y escala tus agentes. Tú le dices qué hacer — él se encarga de correr el modelo, las herramientas y las conversaciones.

Microsoft Agent Framework SDK

Es el SDK open-source de Microsoft (Python / .NET) para construir agentes con más control programático: workflows, middleware, context providers, multi-agente y observabilidad.

⚙️ Arquitectura

El Patrón que se Repite en TODO

Los 9 labs siguen el mismo flujo de 4 pasos. El código crea y registra los agentes directamente en Azure AI Foundry (la nube) — no se ejecutan localmente.

# 1. CONECTAR client = AIProjectClient(endpoint, credential) # 2. CREAR AGENTE agent = client.agents.create_version( agent_name="mi-agente", definition=PromptAgentDefinition( model="gpt-4o", instructions="Eres un asesor...", tools=[...] ) ) # 3. CONVERSAR response = openai_client.responses.create( conversation=conv.id, input="Mi pregunta", extra_body={"agent": {...}} ) # 4. LIMPIAR client.agents.delete_version(...)
01

Conectar

Autenticarse con Azure y apuntar al proyecto en AI Foundry

02

Crear Agente

Se registra en Foundry con instrucciones, modelo y tools

03

Conversar

Crear conversación, enviar mensajes, obtener respuestas

04

Limpiar

Borrar agentes y recursos para no generar costos

☁️ ¿Dónde vive el agente?

El agente no se ejecuta en tu máquina. Cuando llamas create_version(), se crea y registra en Azure AI Foundry. El servicio en la nube hospeda el agente, conecta el modelo LLM, ejecuta las herramientas y maneja las conversaciones. Tu notebook solo envía instrucciones y recibe respuestas.
